<completed_work>

- **Root cause found & fixed**: Next.js Data Cache was caching Neon HTTP responses
  indefinitely — `neon(DATABASE_URL)` without `{ fetchOptions: { cache: 'no-store' } }`
  caused `/api/live` to return stale data forever. Fixed in `lib/db.ts` (commit `9d7f811`).

- **Multi-source dashboard now works**: Live chart correctly shows one line per device/source.
  Confirmed with two simultaneous test sources (device_A, device_B) via the debug pipeline.

- **1-second rolling Leq smoothing**: Mic chart points now use energy-average over 4×250ms
  samples instead of raw instantaneous dB. Reduces measurement jitter without hiding peaks.
  `components/dashboard/live-chart.tsx` — `LEQ_SMOOTH_N = 4` constant (commit `232731c`).

- **Mic recorder on calibration page**: `components/calibration/mic-recorder.tsx` added.
  Compact "Use mic" button + dB readout in calibration page header. Flushes to
  `/api/browser-ingest` so this device appears as an active source within 30s.
  (commit `db8c10c`).

- **Calibration session_id bug fixed**: Neon returns BIGSERIAL ids as strings; `finish`
  route rejected them with `typeof !== 'number'`. Fixed with `Number()` coercion in both
  `start/route.ts` and `finish/route.ts` (commit `1e5f571`).

- **Env vars pulled**: `.env.local` populated from Vercel development environment.
  Contains DATABASE_URL, DATABASE_URL_UNPOOLED, INGEST_SECRET, GTFS_RT_API_KEY etc.
</completed_work>

<decisions_made>

- `cache: 'no-store'` on the neon client (not per-query) — affects all queries globally,

- Rolling Leq window = 4 samples (1 second) — standard for acoustic measurements.
- MicRecorder is a standalone component (not extracted from LiveChart hook) to keep scope
  minimal; full hook refactor deferred.
- Calibration page: mic toggle placed in page header row (flex justify-between with h1).
